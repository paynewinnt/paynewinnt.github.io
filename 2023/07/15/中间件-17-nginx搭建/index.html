配置要点说明

  1. 代理规则：
    • 必须使用与 Ingress 完全匹配的 host 地址（如 nginx.kubezt.com）
    • 直接使用 IP+端口 会导致 404 错误
  2. 路径匹配：
    • 转发到单独服务可行，但转发到 Ingress 需要特殊配置
  3. 包含机制：
    • 使用 include /usr/local/nginx/conf.d/*.conf 加载模块化配置
  4. 负载均衡：
    • 通过 upstream 模块配置多后端服务器
    • 支持权重分配（weight 参数）
